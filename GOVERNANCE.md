# Ombre Governance

## Project Structure

Ombre is an open source project maintained by its founding team with community contributions.

## Decision Making

**Day-to-day decisions** (bug fixes, minor improvements) are made by maintainers directly via pull request review and merge.

**Significant decisions** (architecture changes, new features, breaking changes) are discussed openly in GitHub Issues or Discussions before implementation. Community input is welcomed and considered.

**Strategic direction** is set by the core maintainer team and communicated transparently via GitHub Discussions.

## Maintainers

Core maintainers have merge access and are responsible for the health and direction of the project.

Current maintainers:
- Ombre Core Team ([ombreaiq@gmail.com](mailto:ombreaiq@gmail.com))

## Becoming a Maintainer

Contributors who make sustained, high-quality contributions to the project may be invited to become maintainers. This is based on demonstrated technical ability, reliability, and alignment with the project's values.

## Versioning

Ombre follows [Semantic Versioning](https://semver.org/):
- **MAJOR** — breaking changes to public API
- **MINOR** — new features, backward compatible
- **PATCH** — bug fixes, backward compatible

## Releases

Releases are tagged on GitHub. Release notes are published in [CHANGELOG.md](CHANGELOG.md).

## Licensing
